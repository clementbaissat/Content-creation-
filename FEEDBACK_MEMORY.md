# HopeStage Feedback Memory

This file stores reusable lessons from Clement's review feedback so the content system improves over time instead of repeating the same mistakes.

## How to use this file

- Add only feedback that should change future output behavior.
- Prefer specific, operational rules over vague reminders.
- When possible, encode the lesson in code and docs, not only in prose.
- Review this file before generating social assets, thumbnails, or post drafts.

## Active learnings

### Brand assets

- Always use canonical HopeStage brand assets from `assets/brand/`.
- Do not use ad hoc desktop exports or older local logo files when an approved repo asset exists.
- For light backgrounds, prefer the approved horizontal lockup in `assets/brand/HopeStage logo home.png` or the pale-background lockup in `assets/brand/Frame 10.png`.

### Instagram carousel format

- Instagram carousel slides should default to `1:1` square (`1080x1080`) unless a different format is explicitly requested.
- Do not treat Instagram carousel slides as generic `4:5` portraits.
- Keep logos, kicker text, CTA bars, and main copy inside a generous square safe zone so the Instagram preview UI does not make the layout feel cramped or clipped.

### Visual review discipline

- When feedback mentions a visual mismatch, check the actual rendered output instead of trusting the intended format.
- If a social format has platform-specific behavior, encode it in the generator and the documentation.
